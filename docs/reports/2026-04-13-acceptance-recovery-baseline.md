# 2026-04-13 验收恢复基线

## 当前总体现状

- 本轮 50 题严格验收总分约为 `4.83/10`
- 当前主要问题不是“完全答不出来”，而是“会返回，但理解错、粒度错、表达错、超时多”
- 后续修复应优先围绕五类问题推进：超时、县市粒度、时间解析、代词承接、空数据解释

## 低分问题簇

### 1. 超时

- `04` 今年以来虫情最高的是哪些县？
- `05` 最近两周墒情异常最多的是哪些地区？
- `06` 我问的是县，不是市，过去5个月虫情最高的县有哪些？

**当前症状**
- 均超时约 60 秒
- 无有效 answer
- 无 evidence

**期望行为**
- 必须在受控时间内返回
- 即使回退，也要给出可读结果或澄清，而不是超时空白

### 2. 县市粒度错误

- `07` 常州市下面虫情最严重的县有哪些？
- `08` 苏州市下面墒情异常最多的县有哪些？
- `10` 江苏范围内，虫情最高的是哪些市？再细到县。
- `24` 未来10天哪些县风险最高？

**当前症状**
- 问县答市
- `再细到县` 被当成 `region_name`
- `未来10天哪些县` 被错误抽成 `天哪些县`
- 多处仍泄漏 `1970-01-01`

**期望行为**
- 用户明确说“县”时，最终 route / query / answer 全程保持县级
- 时间解析失败不能伪造 `1970-01-01`

### 3. 占位词与跟进承接不足

- `27` 为什么这个县的墒情异常最多？
- `37` 某设备最近7天触发了几次预警？
- `39` 某县下面有哪些设备出现过异常？
- `40` 给我列出最近最活跃的10台设备。

**当前症状**
- `这个县` 没正确继承上文
- `某设备` / `某县` 被当成真实实体
- “最近最活跃的10台设备”被误路由成“最近一次设备”

**期望行为**
- 缺实体时明确澄清
- 有上下文时优先继承 thread 中的最近实体
- 设备排行和设备详情要分开

### 4. 空数据与未知区域能力不足

- `42` 未知区域对应的是哪些设备？
- `43` 哪些记录的县字段为空？
- `44` 哪些数据没有匹配到区域？

**当前症状**
- `42` 误答“未识别到设备编码”
- `43` 把“县字段为空”误答成 `sms_content` 为空
- `44` 没有给出未匹配区域的数据清单

**期望行为**
- 明确区分字段空值查询、未知区域设备查询、区域映射失败查询
- 返回清单或至少返回正确字段的计数和样例

## 修复优先级

1. 先止血：`04/05/06/07/08/10/24`
2. 再修承接：`27/37/39/40`
3. 再修空数据：`42/43/44`
4. 最后统一回答质量和专家表达

## 本基线的用途

- 作为本轮提分的固定对照
- 后续每修一类问题，都要重新跑 50 题验收，并对比这份基线
- 如果这些问题再次出现，说明语义收口或 planner 约束发生回归
